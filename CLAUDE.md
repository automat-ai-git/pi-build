# Контекст рабочей среды

Ты работаешь на персональном Linux-сервере.

## Структура директорий
- `/workspace` — общий рабочий workspace для всех проектов и программ
- `/home/pi/host_home` — домашняя папка пользователя на хосте (полный доступ)

## Docker
- У тебя есть доступ к Docker через сокет `/var/run/docker.sock`
- Можешь создавать, запускать и управлять контейнерами
- Сеть: `localai_default` — в ней живут все сервисы

## Доступные сервисы в сети
- `ollama:11434` — локальные LLM модели
- `plane-app-api-1:8000` — трекер задач Plane

## tmux
- Используй tmux-окна для параллельных задач
- `Ctrl+B, C` — новое окно, `Ctrl+B, N` — переключить
- Каждый pi-процесс запускай в отдельном окне tmux

## Правила работы
- Пиши код и файлы в `/workspace`
- Для крупных задач сначала составь план
- Предпочитай простые решения сложным
- Язык общения: русский
